# construction — domain pack

> Construction runs in two registers. The **big-site contractor / professional build** is hierarchical, contract-bound, and unambiguous about who carries which risk — reach for it when the point is coordination, obligations over time, and what triggers what at scale. **DIY / homeowner** is one person deciding for themselves: the flat-pack half-assembled in the hall, the shelf that's good enough for the garage, the toolbox in the cupboard — reach for it when the point is a single judgment, a day-off job, or something that quietly accumulates. Both map onto how an AI coding session runs; offer both and let the concept pick.

---

## Register axis

This pack carries two registers. Offer both — the model picks the right one per concept *when both are on the table*.

| Register | Feel | Reach for it when the concept is about… |
|---|---|---|
| **Big-site contractor** (subsystem) | a professional build under contract, many trades in parallel | hierarchy, handoff, who-carries-which-risk, inspections, parallel work, obligations over time |
| **DIY / homeowner** (everyday) | one person deciding for themselves, a job that's good enough for a day out from the usual | a single judgment, something that accumulates, the cheap-and-cheerful path, working with what's in the toolbox |

**Selection cue:** *contractor when the point is coordination and accountability; DIY when the point is one judgment or something that quietly builds up.* When in doubt offer both terms and let the sentence pick — but never mix the two in one breath (no "the site manager checks the flat-pack").

## Roles (who does what)

| Concept | Big-site contractor | DIY / homeowner | Reach for… |
|---|---|---|---|
| User | Developer / client — pays for the build, signs off, holds the land | The homeowner — has to actually live with the result | contractor for sign-off + risk; DIY for "lives with the consequences" |
| Expert (any tier) | Specialist trade / contractor — owns one trade (electrical, structural, mechanical); doesn't reach across | The mate who's handy with electrics — called for that one job | contractor when trades and scope matter; DIY for one-off borrowed help |
| Meta-expert | Architect / lead designer — designs the system; doesn't build it | Whoever sketched the plan on the back of an envelope — set the shape, isn't swinging the hammer | contractor for formal design; DIY for inherited intent |
| Adversarial reviewer | Building inspector / independent surveyor — catches what the trades and architect both missed | A blunt neighbour who says "that wall looks wrong" | contractor for a formal check; DIY for an honest gut read |
| Subagent | Apprentice / labourer — does the prep and the simple runs; doesn't make calls | Roping in a family member to hold the other end | contractor in a managed trade; DIY for a delegated chore |
| Domain expert | Specialist consultant (acoustic / fire / planning) — one narrow technical question, deep expertise | Calling the hardware shop and asking which rawl plug | contractor for a named specialism; DIY for borrowed know-how |
| Sub-expert pair | Lead trade + second opinion, deliberately differing on one axis | Two opinions on whether the shelf bracket is strong enough | either — two voices on one thing, deliberately differing |

|---|---|---|---|
| Persona file | Scope of works / trade contract — what this trade is hired to do, written down | The instruction sheet you actually read before starting | contractor for a formal role; DIY for an ad-hoc reference |
| Calibration card | Spec sheet / building regs reference — the standard the work must meet | The measurements pencilled on the wall before you cut | contractor for a house standard; DIY for a kept mark |
| Instinct file | The trade's toolbox + muscle memory — what fires automatically before conscious thought | Knowing by feel whether the drill is biting properly | either — what fires before you think |
| Notepad / reflex bank | Site notebook / day book — reminders the trade scribbles between rooms | The sticky note on the toolbox | contractor mid-site; DIY for a quick reminder |
| Working memory (post-/dream) | Lessons learned from the last project | "Last time I forgot to turn the water off first" | contractor for project-to-project; DIY for personal habit |
| Hook | Inspection trigger / cert renewal — fires automatically when stage X is reached | The moment you realise you need to check the fuse box | contractor for a formal stage gate; DIY for a personal prompt |
| MCP server | Builders' merchant / supply yard — where materials arrive from; not on-site by default | The cupboard under the stairs — what's already in | contractor for the supply chain; DIY for what's to hand |

## Rhythms (how time and pace work)

| Concept | Big-site contractor | DIY / homeowner | Reach for… |
|---|---|---|---|
| AWE (token effort) | Labour hours + materials — the actual cost of doing the thing | The trip to the DIY shed, the prep, and the tidying up | either — the real cost of doing the thing |
| Session | A day on site / a work package — one bounded run of activity | A self-contained run at the job | contractor for a managed run; DIY for one sitting |
| Session-open | Toolbox talk / pre-start briefing — last reads before the trades pick up tools | Checking what you've got before you start | contractor for a team brief; DIY for the solo check |
| Session-close | End-of-day site clean + handover — tidy, sign off the day, note what's outstanding | Putting the tools away and noting what's still to do | either — tidy, restock, note for next time |
| Calibration cycle | Stage inspection / milestone review — walk the site at first-fix / second-fix / practical completion | Standing back and asking "does this look right?" | contractor for a formal review; DIY for a habit check |
| Coaching pass | Mentoring with a journeyman — targeted feedback on one trade | Watching a tutorial to fix one thing | contractor for targeted feedback; DIY for self-teaching |
| Drift | Scope creep / spec deviation — what's being built isn't what's on the drawings | The shelf that got bigger once you started | contractor for formal deviation; DIY for a job that grew |
| Cross-triage | Independent surveyor visit — outside eyes on the trades' work | Asking a friend who knows more than you to have a look | contractor for a formal outside read; DIY for a second pair of eyes |

## Verbs (what people do)

| Concept | Big-site contractor | DIY / homeowner | Reach for… |
|---|---|---|---|
| Recommend | Submit drawing / RFI — put the proposal in front of the principal | "I reckon we do it this way" | contractor for a formal submission; DIY for a suggestion |
| Veto | Reject / require rework — "this isn't to spec, pull it out" | "No, that's not going up like that" | contractor for a formal rejection; DIY for a hard no |
| Defer | Hold pending inspection / variation order — don't proceed until the question is answered | "Leave it for now, sort it later" | either — wait, don't start yet |
| Approve | Sign off / issue certificate — the work passes, the stage completes | "That'll do, crack on" | contractor for a formal sign-off; DIY for a casual yes |
| Calibrate | Re-measure / re-spec — adjust the drawings to what's actually buildable | Re-measuring before you cut again | either — adjust the mark, the spec, the plan |
| Iterate | Rework / re-fit — strip out and redo | Have another go | either — same job, fix what broke |
| Build (new expert) | Bring on a new trade — award the contract, hand over the scope | Rope in someone who can do what you can't | contractor for a new contracted trade; DIY for borrowed help |
| Retire (an expert) | Decommission / strip out — off the site; that trade's work is gone | The job's done, don't need them again | contractor for closing a contracted line; DIY for finished and gone |
| Adversarial pass | Independent structural review — someone who didn't design or build it | Ask the bluntest person you know to poke at it | contractor for a formal review; DIY for a gut check |
| Stance protocol | Compliance / value engineering / future-maintainability — three lenses every drawing is checked under | "Is it safe, is it worth it, will I regret it?" | either — three questions every job needs |

## Freeform mappings

**Big-site contractor** (coordination, contracts, accountability):

- **The site** — the chat window or the codebase. Everything happens here; outside the hoarding is somebody else's problem.
- **Foundations** — the schema, the base architecture. Get this wrong and everything above moves.
- **First fix** — the rough-in. Wiring, plumbing, structural systems. Done before the walls close.
- **Second fix** — finishes. Switches, taps, paintwork. Cosmetic until you live in it.
- **Snagging** — the drift-detection pass at the end. Catalogue of what's almost right.
- **Variation order** — a scope change with a paper trail. Both sides sign.
- **RFI** (Request for Information) — the open question waiting on the architect.
- **The drawings** — the spec. The single source of truth for what is supposed to exist.
- **Practical completion** — the ship date. Usable, not yet finished.
- **Defects liability period** — post-launch warranty window. Things will still go wrong; the contract says who fixes them.
- **As-built** — what actually exists vs what was drawn. The honest record after drift.
- **The handover pack** — documentation passed to the operator at completion. Drawings, certs, manuals.

**DIY / homeowner** (one person, a day off, good enough):

- **The flat-pack half-assembled in the hall** — a partially-built thing you have to finish before the room is usable. Work in progress that blocks the space.
- **Good enough for the garage** — the standard that clears the bar for this context, even if it wouldn't pass inspection elsewhere. Acceptable POC quality.
- **The toolbox in the cupboard** — everything you actually have to hand, not an ideal set. Working within the tools already loaded.
- **Re-doing it properly later** — a known shortcut taken now, documented as a future fix. Technical debt with full intent.
- **Realising you've drilled into a pipe** — discovering a hidden dependency mid-job. Unverified assumptions hitting reality.
- **The instruction sheet at the bottom of the box** — the reference you didn't read first and now need urgently. Late-discovered spec.

## When the analogy doesn't fit

- **Short-cycle iterative work.** Cooking is faster and more responsive. Construction implies cure times.
- **Pure run-time / live-service problems.** Once a building is up, the analogy gets thin — operations are a different metaphor.
- **Throwaway POCs.** You don't pour foundations for a tent. Reach for cooking or home.
- **Brand / aesthetic / voice problems.** Construction is structural; it doesn't carry feel and finish well.
- **Don't mix the two registers in one breath.** Big-site contractor *or* DIY per concept, never "the site manager checks the flat-pack." Offer both in the pack; pick one in the sentence.
