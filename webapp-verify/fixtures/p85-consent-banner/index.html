About webwitness P8.5

A synthetic fixture used to validate the dismiss_consent live-trigger leg in a controlled environment.

Why this fixture exists

Six wild targets (undavos.com, termly.io, theguardian.com, zeit.de, bbc.com, cookiebot.com) all failed to surface a blocking consent modal in a chrome-devtools-mcp accessibility-tree snapshot. Either CMP scripts detect automation and skip injection, or the modal renders in a top-layer / shadow-DOM that the snapshot does not traverse.

This page eliminates both confounds: the banner is a plain visible <div> element with standard ARIA roles, served from localhost. If the LLM still does not pick the dismiss_consent tactic against this fixture, the bug is in the substrate, not the wild-target environment.

The actual article body — only revealed once the banner is dismissed

This paragraph contains the success-criterion phrase: quokka safari handbook. The webwitness journey requires this exact phrase before declaring PASS, so the LLM cannot match on header copy or marketing text — it must dismiss the banner first to make this content visible.
